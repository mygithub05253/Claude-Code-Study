# 코드베이스 탐색 에이전트 (explore-agent)

## 한 줄 요약

새 프로젝트에 투입되거나 오래된 코드베이스를 다시 볼 때, Explore 에이전트 패턴을 활용해 전체 구조를 빠르게 파악하고 핵심 파일 목록을 자동으로 정리한다.

## 언제 사용하나요?

- 처음 보는 레포지토리를 빠르게 이해해야 할 때 (팀 합류, 오픈소스 기여 시작)
- 리팩토링 전에 의존 관계와 핵심 모듈을 먼저 파악하고 싶을 때
- 레거시 코드베이스에서 특정 기능이 어디 구현되어 있는지 전체 탐색이 필요할 때
- 메인 Claude 세션 컨텍스트를 아끼면서 탐색만 별도로 진행하고 싶을 때

## 핵심 개념

Claude Code의 서브에이전트는 `Task` 도구를 통해 별도의 격리된 컨텍스트에서 실행된다. Explore 에이전트 패턴은 메인 세션이 서브에이전트에게 "이 레포를 탐색하고 요약해줘"라는 명령을 위임하는 구조다.

동작 흐름:
```
메인 Claude 세션
  → Task 도구: "다음 디렉토리를 탐색하고 구조 보고서를 작성해줘: /workspace"
    → 서브에이전트 (독립 컨텍스트)
      → Glob으로 파일 목록 수집
      → Read로 핵심 파일 내용 확인
      → 구조 보고서 작성 후 반환
  → 메인 세션: 보고서를 받아 다음 작업에 활용
```

서브에이전트의 장점:
- **컨텍스트 격리**: 탐색 중 읽은 수백 개 파일이 메인 컨텍스트를 오염시키지 않는다
- **병렬화 가능**: 여러 디렉토리를 동시에 탐색하는 에이전트를 동시에 실행할 수 있다
- **재사용**: 탐색 결과를 파일로 저장하면 이후 세션에서 재활용 가능하다

## 실전 예제

**상황**: Next.js 15 "동아리 공지 게시판" 프로젝트 신규 팀원이 합류하여 전체 구조 파악이 필요한 상황

### Claude Code 세션에서의 실제 사용 방법

메인 세션에서 다음과 같이 요청한다:

```
이 프로젝트의 전체 구조를 탐색하는 서브에이전트를 실행해줘.
에이전트가 수행할 작업:
1. 디렉토리 트리 구조 파악 (apps/, packages/, content/, scripts/)
2. 각 디렉토리의 역할과 주요 파일 식별
3. package.json에서 의존성과 스크립트 확인
4. 진입점 파일(index.ts, main.ts, page.tsx 등) 내용 확인
5. 결과를 docs/codebase-map.md 파일로 저장
```

Claude가 내부적으로 Task 도구를 사용하여 서브에이전트를 실행한다:

```
[Task 도구 실행]
프롬프트: 주어진 프로젝트 디렉토리를 탐색하여 구조 보고서를 생성하라.
          최종 결과는 docs/codebase-map.md에 저장하라.
디렉토리: /workspace (claude-code-study 프로젝트)
```

### 서브에이전트가 생성하는 `docs/codebase-map.md` 예시

```markdown
# 코드베이스 구조 맵 (2026-04-12 기준)

## 디렉토리 역할
| 디렉토리 | 역할 |
|---|---|
| apps/docs/ | VitePress 문서 사이트 |
| packages/parser/ | SKILL.md → JSON 파서 |
| packages/explainer/ | JSON → 한국어 해설 MD |
| content/ko/ | 한국어 해설 콘텐츠 |
| scripts/ | 파이프라인 스크립트 |

## 핵심 파일
- apps/docs/.vitepress/config.ts — VitePress 설정
- packages/parser/src/index.ts — 파서 진입점
- scripts/sync-content-to-docs.ts — 콘텐츠 동기화

## 의존성 (주요)
- vitepress: 1.x (문서 사이트)
- gray-matter: 프론트매터 파싱
- zod: 스키마 검증
- fast-glob: 파일 패턴 검색
```

### 탐색 에이전트 재사용 프롬프트 템플릿

```
서브에이전트를 실행해서 다음 탐색 작업을 수행해줘:

탐색 대상: [디렉토리 경로]
탐색 목적: [무엇을 파악하고 싶은지]
출력 파일: [결과를 저장할 파일 경로]

탐색 단계:
1. [첫 번째 탐색 단계]
2. [두 번째 탐색 단계]
3. 결과를 지정한 출력 파일에 마크다운으로 저장
```

## 학습 포인트

- **서브에이전트 = 독립 컨텍스트**: 서브에이전트는 메인 세션과 메모리를 공유하지 않는다. 결과를 메인 세션으로 가져오려면 파일로 저장하거나 Task 도구의 반환값을 활용해야 한다.
- **탐색 범위를 명확히 지정**: "전체 프로젝트 탐색"처럼 범위가 너무 넓으면 에이전트가 과도하게 많은 파일을 읽어 느려진다. "apps/docs/ 디렉토리의 VitePress 설정만"처럼 범위를 좁히는 것이 효율적이다.
- **결과 캐싱 전략**: 탐색 결과를 `docs/codebase-map.md`에 저장하고 git에 커밋하면, 다음 세션에서 동일한 탐색을 반복하지 않아도 된다. 코드 변경 시에만 재탐색한다.
- **병렬 탐색 활용**: 프론트엔드와 백엔드가 분리된 경우, 두 개의 서브에이전트를 동시에 실행하여 각각을 탐색하게 하면 시간을 절반으로 줄일 수 있다.
- **컨텍스트 절약 효과**: 대형 레포를 탐색하면 수십 개 파일을 읽어야 하고, 이 내용이 모두 컨텍스트에 들어온다. 서브에이전트에게 위임하면 메인 컨텍스트는 요약 결과만 받는다.

# Agents 패턴 해설

> Claude Code의 **Sub-agent / Multi-agent 패턴** 을 대학생 프로젝트 맥락에서 설명합니다.

## 이 섹션의 목적

- 단일 에이전트 / 서브 에이전트 / 멀티 에이전트의 차이를 명확히 합니다.
- 각 패턴의 사용 상황, 프롬프트 체인, 에러 복구 전략을 한국어로 정리합니다.
- 공식 내장 에이전트와 커뮤니티 에이전트를 선별해 소개합니다.

## 향후 해설 대상 (Phase 5)

- Explore 에이전트 — 코드베이스 탐색 자동화
- Plan 에이전트 — 구현 계획 수립 분리
- 병렬 에이전트 디스패치 패턴
- GSTACK 역할 기반 에이전트 (커뮤니티)
